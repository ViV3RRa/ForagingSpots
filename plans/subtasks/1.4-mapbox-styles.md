# Subtask 1.4 — Custom Mapbox styles (light + dark)

## Context

You are implementing one subtask of the "Skovens Skatte" UI redesign migration. Before starting:

1. Read `plans/new-design-migration.md` — the master plan (design language, token tables, scope, screen-to-component map). This file is **Phase 1, subtask 1.4**.
2. Read `CLAUDE.md` for project conventions and commands.

The app renders its map with Mapbox GL via react-map-gl (`src/components/MapView.tsx`, token/config in `src/utils/mapbox.ts`). The design's earthy basemap is central to the redesign. The decision (see master plan) is to use **custom Mapbox Studio styles** for light and dark.

**Depends on:** subtask 1.1 (theme context, for style switching).

## Task

1. **Config plumbing (code, do this fully):**
   - Add `VITE_MAPBOX_STYLE_LIGHT` and `VITE_MAPBOX_STYLE_DARK` env vars, read in `src/utils/mapbox.ts` with fallbacks to reasonable stock styles (e.g. `mapbox://styles/mapbox/outdoors-v12` light, `mapbox://styles/mapbox/dark-v11` dark) so the app works before the Studio styles exist.
   - Export a `getMapStyle(theme)` helper.
   - `src/components/MapView.tsx` and `src/components/LocationPickerModal.tsx`: select the style from the active theme (ThemeContext from 1.1) and switch live when the theme changes (react-map-gl `mapStyle` prop change; verify markers survive the style swap).
   - Update `.env.example` / README env docs if present.
2. **Offline caching:** check the Workbox `runtimeCaching` rules in `vite.config.ts` — ensure the patterns match `api.mapbox.com` style + tile + sprite + glyph URLs for the new styles (they are same-origin patterns as today; extend if the rule is style-specific).
3. **Mapbox Studio (manual step — document, don't fake):** you cannot create Studio styles yourself. Produce a concise instruction file `plans/subtasks/1.4-mapbox-studio-instructions.md` for the user: start from Monochrome (or Outdoors), set land/background/water/roads/paths to the exact light and dark `--map-*` hex values you extracted from the design, reduce POI/label density, publish, and paste the two style URLs into `.env`. List each layer-group → color mapping explicitly.

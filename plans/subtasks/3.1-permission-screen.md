## Task

1. New component `src/components/LocationPermissionScreen.tsx` matching the design (both themes, safe areas).
2. **Flow wiring** (in `App.tsx` / `MainMapScreen.tsx`):
   - After sign-in, before/over the map, show the screen **once** when geolocation permission state is `prompt` (use `navigator.permissions.query({name:'geolocation'})` where available; treat unsupported browsers as `prompt`).
   - "Tillad placering" → start the `useUserLocation` watcher (this triggers the native prompt), then continue to the map regardless of grant/deny.
   - "Ikke nu" → continue to the map; location features (distance, locate button, follow) degrade gracefully as they already do.
   - Persist an "asked" flag in localStorage so the screen never nags again; if the user later taps the Locate button while permission is still `prompt` and the flag exists, just trigger the native prompt directly (no priming screen).
3. Never block the app: any error path lands on the map.

## Acceptance criteria

- `npm run build` + `npm run lint` pass.
- Fresh profile (permission `prompt`, no flag): screen shows once after sign-in; both actions proceed to the map; the native prompt only fires on "Tillad placering".
- Already granted/denied or previously asked: screen never shows.
- Screen matches the design in both themes.

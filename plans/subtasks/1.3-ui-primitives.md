# Subtask 1.3 — Base UI primitives restyle

## Context

You are implementing one subtask of the "Skovens Skatte" UI redesign migration. Before starting:

1. Read `plans/new-design-migration.md` — the master plan (design language, token tables, scope, screen-to-component map). This file is **Phase 1, subtask 1.3**.
2. Read `CLAUDE.md` for project conventions and commands.

The app is a Danish foraging-spots PWA (React 19 + TypeScript + Vite, Tailwind CSS, shadcn/ui). All Phase 2/3 screen subtasks compose the primitives you build here — restyle them once so screens don't repeat one-off styles.

**Depends on:** subtasks 1.1 (tokens/theme) and 1.2 (typography).

## Design source of truth (look this up — do not work from memory)

- Link: https://claude.ai/design/p/dd75eb78-ea23-46b0-ba4b-7aeb33aff7e4?file=Skovens+Skatte.dc.html
- Fetch with the claude-design MCP tool: `mcp__claude-design__read_file`, `project_id: "dd75eb78-ea23-46b0-ba4b-7aeb33aff7e4"`, `path: "Skovens Skatte.dc.html"`. Content is HTML-entity-escaped and large; read in chunks as needed.
- **For this subtask, survey the whole mock** and extract the recurring component recipes with exact values:
  - **Primary button:** ~56px height, 16px radius, `var(--accent)` bg, `var(--accent-ink)` text, Spectral 600 ~17px, glow shadow `0 8px 22px -6px var(--accent)` (see "Kom i gang", "Log ind", "Gem fund", "Vis resultater", "Bekræft placering").
  - **Secondary/outline button:** surface bg, `--line` border, 14px radius (see "Tilføj foto", "Annullér").
  - **Text/ghost button:** plain `--ink2` text (see "Jeg har allerede en konto", "Ikke nu").
  - **Icon buttons:** 36–52px circles/rounded-squares on surface with `--line` border (back buttons, close buttons, filter button).
  - **Inputs:** 54px height, 14px radius, surface bg, `--line` border, Spectral 16px value text; uppercase Space Mono field labels above (sign-in screen).
  - **Bottom sheet chrome:** 28px top radius, centered 42×5px grab handle in `--line`, scrim `rgba(20,15,8,.4)` with `ss-fade`, sheet enters with `ss-rise` (detail drawer, add sheet, filter sheet, install sheet).
  - **Type badge:** circular gradient badge with PNG icon, `--pin-ring` border (3–4px), inner highlight `inset 0 0 0 2-3px rgba(255,255,255,.3)`, drop shadow — used at 44/52/60/72px (map pins, filter rows, list rows, detail header).
  - **Mono label:** Space Mono ~11px, uppercase, letter-spacing .1–.12em, `var(--mono)` color.

## Task

1. **Buttons** — update `src/components/ui/button.tsx` variants (or add new ones) for primary/secondary/ghost/icon per the recipes above. Keep the shadcn API (`variant`, `size`) so existing call sites still compile.
2. **Inputs** — update `src/components/ui/input.tsx`, `textarea.tsx`, `label.tsx` to the new field style.
3. **Sheet primitive** — standardize one bottom-sheet primitive (extend shadcn `sheet.tsx` or `drawer.tsx`, pick whichever the codebase already leans on — check `PinDetailsDrawer.tsx`) that renders the design's sheet chrome (radius, handle, scrim, animations). Later subtasks (2.6, 2.7, 2.8, 3.3) will adopt it; do not migrate them yet.
4. **`TypeBadge` component** (new, e.g. `src/components/TypeBadge.tsx`): props `{ type: ForagingType, size: number }`; renders the circular gradient badge using `getForagingSpotConfig` from `src/components/icons/index.tsx` (gradients + PNG paths already exist there and match the design).
5. **`MonoLabel` component** (new, e.g. `src/components/ui/MonoLabel.tsx`) or a `.label-mono` class if 1.2 already added one — one canonical implementation.
6. **Complete Danish labels:** `src/utils/danishLabels.ts` covers only 12 of the 25 `FORAGING_TYPES` (see `src/components/types.ts`). Add correct Danish names for the rest (bay_bolete → Brunstokket rørhat, black_currant → Solbær, black_trumpet → Trompetsvamp, blackberry → Brombær, cranberry → Tranebær, field_mushroom → Markchampignon, hedgehog_mushroom → Pigsvamp, parasol_mushroom → Parasolhat, raspberry → Hindbær, red_currant → Ribs, sheathed_woodtuft → Foranderlig skælhat, sloe → Slåen, wild_strawberry → Skovjordbær — verify against existing label style/casing in the file).
7. **Showcase:** repurpose `src/components/IconShowcase.tsx` (dev-only component) to render the new primitives in both themes for visual verification.

# Subtask 2.12 — Map error state

## Context

You are implementing one subtask of the "Skovens Skatte" UI redesign migration. Before starting:

1. Read `plans/new-design-migration.md` — the master plan (design language, token tables, scope, screen-to-component map). This file is **Phase 2, subtask 2.12**.
2. Read `CLAUDE.md` for project conventions and commands.

The map lives in `src/components/MapView.tsx` inside the chrome of `MainMapScreen.tsx`. Today the
only error handling is a hard-coded English red screen shown when the Mapbox token is missing
(`validateMapboxToken()` → `mapError` state, around line 197). There is **no** handling of runtime
map failures (style/tile load errors, e.g. offline first-load or an invalid token rejected by the
API). The design adds a dedicated error card for "failed load / missing token".

**Depends on:** subtasks 1.1–1.3 (tokens, fonts, button primitives) and 2.3/2.4 (chrome + map view), all done.

## Task

1. Replace the red English error screen in `MapView.tsx` with the design's error card (both
   themes). Map the actual error reason into the Space Mono footer line (e.g. missing token vs.
   load failure).
2. Extend error detection beyond the missing-token check: handle runtime map failures via
   react-map-gl's `onError` (style/tile fetch failures — but do **not** treat transient tile
   errors while browsing an already-loaded map as fatal; only fail before/at first successful
   style load).
3. **"Prøv igen"** resets the error state and remounts/reloads the map (for the missing-token
   case it may immediately re-fail — that's fine, the card just re-renders).
4. **"Vis fund som liste"** switches to list view — the `viewMode` state lives in
   `MainMapScreen.tsx`, so expose a callback prop from `MapView` (or lift the error state).
5. Hide the FAB and location chip while the error shows (both rendered by `MainMapScreen`, so the
   error state must be visible there — a `onMapErrorChange` callback or lifted state), and the
   locate button if it renders inside `MapView`. Keep top bar and view toggle.
6. Danish copy throughout per the design; no English leftovers.

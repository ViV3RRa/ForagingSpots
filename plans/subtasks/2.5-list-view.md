## Task

1. **Rows:** rebuild the spot rows per the design (badge via `TypeBadge`, typography, dividers). Skip the latin-name line. Keep tap → detail drawer, and keep existing search/sort functionality (restyle the sort control with 1.3 primitives).
2. **Relative dates:** add a small Danish relative-date formatter (e.g. in `src/utils/`) producing the design's compact forms ("2 d.", "5 d.", "1 u.", "3 u.", falling back to a short date for older). Use the spot's found/created date as currently displayed.
3. **Distance-to-spot (new feature):**
   - Create `src/utils/distance.ts`: haversine distance + Danish formatting (`0,8 km`, `< 1 km` granularity in meters if you prefer — match the design's "0,8 km" style with comma decimal).
   - Create a shared `useUserLocation` hook (e.g. `src/hooks/useUserLocation.ts`) by extracting/reusing the geolocation watching already done in `MapView.tsx`, so map, list, and detail drawer share one position source instead of multiple watchers. Refactor `MapView` to consume it.
   - Show the distance in each row (placement per design: the design uses `dist` in the detail; in the list you may place it where the coordinates line is or alongside — follow the design's list row exactly; if the mock's list row shows coordinates, show coordinates and add distance only where the design does).
   - Degrade gracefully: no permission / no fix → simply omit distance.
4. **Empty state:** implement the design's empty view when the user has no spots (after filters/search show-nothing states, keep whatever exists but restyle). CTA opens the add flow via the existing callback.
5. Both themes + safe areas.

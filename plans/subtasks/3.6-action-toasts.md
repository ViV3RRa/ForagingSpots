# Subtask 3.6 — Action toasts (success / error)

## Context

You are implementing one subtask of the "Skovens Skatte" UI redesign migration. Before starting:

1. Read `plans/new-design-migration.md` — the master plan (design language, token tables, scope, screen-to-component map). This file is **Phase 3, subtask 3.6**.
2. Read `CLAUDE.md` for project conventions and commands.

The app uses **sonner** for feedback toasts. The `<Toaster>` is mounted in `src/main.tsx` with
`position="top-right" richColors` (imported directly from `sonner`; the shadcn wrapper
`src/components/ui/sonner.tsx` imports `next-themes` and appears unused — verify and either adopt
or remove it). Toast call sites live mainly in `src/hooks/useForagingSpots.ts` and the offline
sync path (grep for `from 'sonner'`); several messages are still English ("Failed to update
foraging spot") or use the old "skat" wording.

**Depends on:** subtasks 1.1–1.2 (tokens, fonts). Theme comes from the app's own `useTheme`
(`src/hooks/useTheme.ts`), not next-themes.

## Task

1. Restyle sonner to render the design's toast cards for success and error (via `toastOptions`
   class overrides, `unstyled` + custom classes, or `toast.custom` wrappers — pick the least
   invasive approach that keeps the existing `toast.success/error/info(msg, {description})` call
   sites working; `info` can share the success/neutral treatment).
2. Move the Toaster to **bottom-center** with the design's ~110px bottom offset (respect
   safe-area insets) and set the default duration to 3200ms.
3. Wire the toast theme to the app's `useTheme` so dark mode renders correctly. **Dark-mode
   caveat:** the design hard-codes cream text on `--brand`, but `--brand` is gold `#c9a24b` in
   dark — if contrast is poor in practice, use `--brand-ink` for the text/icon instead and note
   the deviation.
4. Align toast copy with the design and the migrated screens' terminology ("fund", not "skat";
   Danish everywhere): e.g. "Fund gemt", "Fund opdateret", "Fund slettet",
   "Kunne ikke gemme — prøv igen". Fix the remaining English error toasts.
5. Descriptions (second line, e.g. the offline "Synkroniseres når forbindelsen genoprettes")
   should render as a smaller Work Sans subline like the design's update-toast subline.
6. Both themes; toasts must not overlap the FAB/toggle or sit under sheets awkwardly (design
   uses z-index above sheets).

- All toast copy is Danish and uses "fund" terminology.

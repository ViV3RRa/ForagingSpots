# Subtask 4.3 — Spot-type asset optimization

## Context

You are implementing one subtask of the "Skovens Skatte" UI redesign migration. Before starting:

1. Read `plans/new-design-migration.md` — the master plan. This file is **Phase 4, subtask 4.3**. It can run any time after Phase 1 but is scheduled late to avoid churn.

**Design lookup is not needed** (no visual changes — same images, smaller files). The design link is in the master plan if needed.

## Problem

`public/spot_types/*.png` — 24 spot-type illustrations, ~600 KB–1 MB each, ~19 MB total — are displayed at ≤72 px (map pins 52px, list badges 60px, detail header 72px, type grid tiles, filter rows 44px; largest usage is the welcome-screen cluster at 76px and lightbox placeholder ~150px). They bloat the PWA precache and first load massively.

## Task

1. Determine the actual maximum rendered size across the app (grep `TypeBadge`/`getForagingSpotConfig` usages) and account for 2–3× devicePixelRatio — target roughly **192–256 px** square output.
2. Resize + compress all 24 PNGs (keep transparency). Use available tooling — `sips` (preinstalled on macOS), or `sharp`/`squoosh` via a small script (e.g. `scripts/optimize-spot-images.mjs`). Consider also emitting WebP with PNG fallback only if the code change is trivial; otherwise plain optimized PNG is fine.
3. Keep the same file names/paths (`public/spot_types/<type>.png`) so no code changes are needed; back up originals outside the repo or rely on git history.
4. Visually verify each type at all badge sizes in the running app (icon showcase from subtask 1.3 helps) — no visible quality loss, no clipped edges.
5. Confirm the Workbox precache total drops accordingly (`npm run build`, check generated manifest size).

## Files

- `public/spot_types/*.png` (replaced)
- Optional: `scripts/optimize-spot-images.mjs`

## Acceptance criteria

- Total `public/spot_types/` size reduced by >85% (expect ~1–2 MB or less).
- All 24 images look crisp at every rendered size on a 3× display setting.
- `npm run build` passes; precache manifest reflects the smaller assets.

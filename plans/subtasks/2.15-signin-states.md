## Task

1. Add the design's `ss-spin` keyframe to `src/styles/tokens.css` (and as a Tailwind animation
   alongside `ss-pulse`/`ss-rise`/`ss-fade`, matching how those are registered).
2. Restyle the sign-in error box to the design card and switch the copy to Danish (wrong
   credentials → design copy; unexpected errors → a sensible Danish equivalent, e.g. "Noget gik
   galt. Prøv igen.").
3. Implement the loading CTA state: spinner + "Logger ind…", button at design opacity, disabled
   while pending (keep the existing disabled logic).
4. Apply the same error/loading recipe to `SignUpScreen.tsx` (it uses the same 2.2 recipe; keep
   its wiring status as-is).
5. Both themes.

## Acceptance criteria

- `npm run build` + `npm run lint` pass.
- Wrong credentials show the design's error card (both themes); submitting shows the spinner
  CTA until the request settles.
- All auth feedback copy is Danish.
- Successful sign-in flow unchanged.

# Subtask 3.10 — Bottom nav bar & map-chrome rework

## Context

You are implementing one subtask of the "Skovens Skatte" UI redesign migration. Before starting:

1. Read `plans/new-design-migration.md` — the master plan (design language, token tables, scope, screen-to-component map). This file is **Phase 3, subtask 3.10** (added 2026-07-09).
2. Read `CLAUDE.md` for project conventions and commands.

The design reworked the map screen's chrome (change request 2026-07-09): a **floating bottom
navigation bar** with a cradled center add-button replaces the bottom-center Kort/Liste pill and
the corner FAB; the satellite layers toggle (3.9) moves to the **bottom-left** where the location
chip used to be; the **location chip is removed** entirely; all edge margins tighten to a
consistent **16px**; the no-location badge (3.8) moves up under the search field and adopts the
offline badge's solid colors; and the **locate button stays visible with no location**.

**Decisions (agreed 2026-07-09):**

- **Map-error state hides the whole bottom bar** (design's `showBottomBar`), including the add
  button. This supersedes the 2.12/3.8 deviation that kept the FAB on the error screen — the
  error card's "Vis fund som liste" is the path to the list, where the bar (and add) lives.
  Remove the deviation comments in `MainMapScreen.tsx`.
- **Satellite mode stays session-only** (3.9 decision stands — plain state, no localStorage; the
  change request's "persist/track the choice in map state" means the mock's component state).
- **Locate with no fix = retry + feedback.** Tapping the locate button while there is no
  position re-invokes `startUserLocation()` (re-fires the permission prompt / GPS attempt) and
  shows a locating state on the button while `useUserLocation().status === 'locating'` (e.g.
  pulse the crosshair with the existing `ss-pulse`, or an equivalent subtle treatment) so the
  tap doesn't feel dead. When status resolves to `available` the map flies to the fix (existing
  behavior); when it stays `unavailable` the amber badge remains.
- **Satellite loading background `#04070e` is not in the mock** — it comes from the change
  request text. Implement it as the map container's background while in satellite mode (see
  Task 5).
- The mock uses fixed margins on a 390px frame; implement edges as `max(16px, safe-area-inset-*)`
  and bottom offsets as `calc(env(safe-area-inset-bottom, 0px) + Npx)`, consistent with the
  existing chrome.

**Depends on:** 2.3/2.4 (chrome, map view), 2.12 (map error), 2.14 (compass), 3.2 (offline
banner), 3.8 (no-location state), 3.9 (satellite mode). **Supersedes pieces of** 2.3 (pill
toggle, FAB placement), 2.12/3.8 (FAB-on-error deviation), 3.8 (locate-button removal, badge
position/colors), 3.9 (toggle position).

    transform:translate(-50%,-50%)` — half above the bar in the notch. `background:

    glyph (mock: text `+` at 30px weight 300 — an SVG plus with equivalent visual weight is
    fine), `box-shadow: 0 8px 20px -4px var(--accent)`, and **`border: 4px solid var(--bg)`** —
    the cradle halo ring in the page-background color. Opens the add sheet; `aria-label`
    "Tilføj fund".
  - **Visibility**: `showBottomBar = !(view === 'map' && mapError)` — the bar shows on the list
    view always and on the map view except the map-error state. It is *not* gated on sheets
    (sheets overlay it, as they do today).
  - **Shape implementation (decided 2026-07-09): CSS mask, not the mock's SVG.** The mock's
    `preserveAspectRatio="none"` SVG stretches the notch on screens wider than the 390px frame
    while the 88px spacer and 60px button stay fixed, so the button would drift out of the
    curve. Instead build the bar as a plain div (`height:66px; border-radius:26px;
    background:barFill`) and cut the notch with a mask anchored at 50% width:

    ```css
    .bar-shadow-wrap { filter: drop-shadow(0 8px 20px rgba(20,15,8,.28)); }
    .bar {
      mask-image: radial-gradient(circle 38px at 50% -14px, transparent 37.5px, black 38.5px);
      -webkit-mask-image: radial-gradient(circle 38px at 50% -14px, transparent 37.5px, black 38.5px);
    }
    ```

    The drop-shadow lives on a wrapper element (filters see the masked alpha, so the shadow
    follows the notch). The mock's notch is this same 38px arc with slightly eased shoulders;
    the circular cutout is visually indistinguishable once the button + halo sit in it. This
    keeps the notch a fixed 38px scoop centered under the button at any screen width — no
    width cap, no JS. Tune the `-14px` center / stop offsets against the mock if the scoop
    looks too deep or shallow. The SVG path above stays as geometry reference only.
  - **Floating buttons** (all 52px circles, z-5 in the mock, unchanged styling from 3.9/2.4/2.14):
    layers toggle `left:16px; bottom:112px`; locate `right:16px; bottom:108px`; compass
    `right:16px; bottom:168px`. Outer edges align with the bar's 16px margins. (The 4px
    layers/locate bottom asymmetry is in the mock — keep the mock's values verbatim.) The bar
    top sits 88px from the bottom edge (22+66), so these clear it by ~20px.
  - **Location chip is gone**: the `current-location badge removed; satellite button now sits
    here` comment marks the spot. Delete the `LocationChip` component and its coordinate
    formatting — no replacement.
  - **Top bar**: 16px effective side margins (mock: `padding:0 6px` wrapper + `0 10px` row)
    instead of the current 24px. Field/button sizes unchanged.
  - **No-location badge**: horizontally centered, `top: noLocTop` = **112px** normally (same
    slot as the offline banner, i.e. directly below the search field) or **182px** when the
    offline banner is showing (banner at top 112, badge stacks below it). Colors now
    **identical to the offline palette**: `noLocBg` `#f6ecd4` light / **solid `#33301f`** dark
    (the translucent `rgba(201,162,75,.16)` dark value is obsolete), border `#e2cf9a`/`#5c5330`,
    ink `#7a5a1e`/`#e8dcae`. Badge markup otherwise unchanged (crossed-pin icon, Space Mono

  - **Locate button**: gated only on `isMapReady` in the mock — visible with no location

## Task

1. **Bottom nav bar.** New component (e.g. `BottomNavBar` in `MainMapScreen.tsx` or its own
   file): the mask-notched bar + Kort/Liste items + cradled add button per the key facts, with
   `bottom: calc(env(safe-area-inset-bottom, 0px) + 22px)` and `left/right: max(16px,
   env(safe-area-inset-*))`. Wire Kort/Liste to `viewMode` and the add button to
   `setShowAddModal(true)`. Render it whenever `!(viewMode === 'map' && mapError)`. The
   `barFill` dark value `#20301c` is hardcoded in the mock — add it as a token or inline
   dark-variant class, matching how other one-off design colors are handled.
2. **Remove the old chrome.** Delete the `ViewToggle` pill and the `FloatingActionButton`
   usage from `MainMapScreen.tsx` (delete `FloatingActionButton.tsx` if nothing else imports
   it), and delete the `LocationChip` component. Remove the now-obsolete "FAB stays on
   map-error" deviation comments. Check `SpotListView`'s bottom padding still clears the new
   bar (bar top overhang: 22 + 66 + 30px of button ≈ 118px + inset).
3. **Reposition the floating buttons.** Layers toggle (in `MainMapScreen.tsx`) moves to
   `left: max(16px, env(safe-area-inset-left))`, `bottom: calc(env(safe-area-inset-bottom,0px)
   + 112px)`; its `isMapReady` gating and colors are unchanged from 3.9. Locate and compass
   (in `MapView.tsx`) move to `right: max(16px, env(safe-area-inset-right))` with bottoms 108px
   and 168px respectively.
4. **Locate button in the no-location state.** Drop the `locationStatus !== 'unavailable'`
   render gate. On tap with no position: `startUserLocation()` (already called today) + the
   locating feedback described in Decisions while `status === 'locating'`.
5. **Satellite loading background.** While `mapMode === 'satellite'`, set the map container's
   background to `#04070e` (e.g. via the `<Map style={{...}}>` prop in `MapView.tsx`) so
   tile-loading gaps read near-black instead of cream; base mode keeps the current background.
6. **Top bar margins.** In `TopBar.tsx`, change the row padding from `max(24px, safe-area)` to
   `max(16px, safe-area)` on both sides.
7. **No-location badge.** Move it from the bottom chip slot to centered below the search field:
   `top: calc(max(14px, env(safe-area-inset-top)) + 70px)` normally (the offline banner's slot —
   mock 112px), or `+ 140px` (mock 182px) when the device is offline so it stacks under the
   banner. Use `useNetworkStatus` (the hook behind `OfflineBanner`) for the offline check. Update

   `--offline-bg` in both themes, so fold it into that token and drop `--noloc-bg` (fix the
   comment that documents the old translucent value).
8. **Scope guard.** List view keeps its own toolbar/empty states; sheets, detail drawer, map
   error card, offline banner, and the satellite style itself are unchanged. No data-layer
   changes.

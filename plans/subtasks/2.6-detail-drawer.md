## Task

1. Rebuild `PinDetailsDrawer.tsx` on the 1.3 sheet primitive with the design's chrome and scroll behavior.
2. **Header:** `TypeBadge` 72px + Danish type label as title (via `getDanishLabel`). No latin line.
3. **Gallery:** design shows exactly 3 tiles; the app allows a variable number of images. Implement: 0 images → single dashed "+" tile; 1 → large tile + "+" tile; 2 → large + small + "+"; 3+ → design layout with count chip ("N fotos"). Tiles show real thumbnails (`getSpotImageThumbnailUrls`), tapping opens the lightbox (`ImageViewer`), "+" triggers the existing add-image flow (`ImageCapture` — check how it's currently invoked from the drawer/edit flow).
4. **Meta rows:** Koordinater (formatted like the design: `56.1234° N · 10.2038° E`) with accent "Redigér ›" opening the location editor (currently `LocationPickerModal`; it gets its own redesign in 2.9 — just keep the wiring); Fundet (Danish date format like "2. okt 2026"); Afstand (haversine util + `useUserLocation` from 2.5; omit the row when position unavailable).
5. **Notes** section with MonoLabel header and Spectral body.
6. **Actions:** primary wide button = Edit (brand bg per design's primary slot); square icon buttons for Delete (accent-colored trash, opens confirm — subtask 2.11 restyles it) and Share (keep the existing share/unshare UI reachable — restyle its inputs/rows with 1.3 primitives; if it doesn't fit the action row, keep it as a section below, tokens-restyled).
7. Keep the pending-sync badge (restyled with new tokens) and all existing behavior: optimistic updates, offline pending state, share validation.

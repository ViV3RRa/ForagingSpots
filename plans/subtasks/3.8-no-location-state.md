# Subtask 3.8 — No-location state (badge + degraded chrome)

## Context

You are implementing one subtask of the "Skovens Skatte" UI redesign migration. Before starting:

1. Read `plans/new-design-migration.md` — the master plan (design language, token tables, scope, screen-to-component map). This file is **Phase 3, subtask 3.8** (added 2026-07-09).
2. Read `CLAUDE.md` for project conventions and commands.

Today the app degrades **silently** when it has location permission but no position fix (and
likewise when permission is denied): `useUserLocation` (`src/hooks/useUserLocation.ts`) only
`console.warn`s errors and exposes `{ position, initialized }` — no error/status surface. The
consequences, verified 2026-07-09:

- Map stays on the Denmark-center fallback with no user dot — indistinguishable from "still locating".
- The location chip (`MainMapScreen.tsx`) is simply not rendered.
- The locate button (`MapView.tsx`) is a **silent no-op**: `flyTo` is skipped and the active
  styling never applies, so tapping it does nothing visible.
- The FAB is a **dead button**: the add sheet is gated on `showAddModal && currentPosition`
  (`MainMapScreen.tsx`), so with no position, tapping + renders nothing.
- List/detail distance lines are omitted (this part is fine — keep it).

**Depends on:** subtasks 1.1–1.3 (tokens, fonts, primitives), 2.3/2.4 (app chrome, map view).

## Task

1. **Expose the condition.** Extend `useUserLocation` so consumers can tell "no position and none
   coming" apart from "still initializing": e.g. return a `status` alongside `position`
   (`'locating' | 'available' | 'unavailable'`), where `unavailable` covers geolocation errors
   (denied, position-unavailable, timeout) and unsupported browsers. Nuance to handle: when the
   permission gate never opened (state `prompt`, user skipped the priming screen), "tjek din GPS"
   is misleading — decide at implementation whether the badge also shows then (simplest, matches
   the design's single badge) or only once a fix was actually attempted; note the choice in code.
2. **No-location badge.** On the map view, when status is `unavailable`, render the design's
   centered amber badge instead of the location chip (new small component or a variant inside
   `MainMapScreen.tsx` next to `LocationChip`). Same vertical position as the chip; hidden while
   the map-error card shows (sheets overlay it anyway). When a fix arrives later, it swaps back
   to the chip live.
3. **Remove the locate button** in `MapView.tsx` while status is `unavailable` (render nothing,
   per the design gate). It must reappear if a position shows up.
4. **Un-dead the add flow — no-location Placering state.** Drop the `currentPosition` gate on
   the add sheet in `MainMapScreen.tsx`: tapping **+** always opens `AddEditModal` (its
   `coordinates` prop becomes nullable). Inside the sheet, implement the design's `addLocEl`
   states:
   - No location → the amber warning card + "Tilføj lokation manuelt" brand button (opens the
     existing location editor, same as editing a find's location). **"Gem fund" is disabled**
     (dimmed per the design, no-op) until a location exists — either arriving automatically
     (GPS fix lands while the sheet is open → pill shows "Nuværende") or picked manually in the
     editor (→ pill shows "Manuel", no pulse on the dot).
   - Backing out of the editor without confirming returns to the sheet still in the warning
     state, save still disabled.
   - A GPS fix arriving later must **not** overwrite a manually picked location — auto-fill only
     applies while the section is still in the warning state (matches the design's `addLoc`
     state machine, where `manual` is terminal).
   - **Editor pin type:** the center pin shows the currently selected type from the type grid —
     the pre-selected Kantarel default stays (decision 2026-07-09) — falling back to the "Andet"
     badge if no type is set.
   - Editing an existing find is unaffected (it always has coordinates).
5. Both themes; Space Mono text; no layout jump when chip ⇄ badge swap (same row/height class).

  ("Nuværende") and save enables without user action.
- With a working location: chip and locate button render exactly as before; badge never appears.
- Badge hidden on the map-error card; list view unaffected (distances still just omitted).
- Correct colors in both themes (amber tint light, translucent gold dark).

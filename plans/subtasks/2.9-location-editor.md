## Task

1. Rebuild `LocationPickerModal.tsx` as a fullscreen view: react-map-gl map (themed style from 1.4, initial center = current spot/GPS coordinates) fills the viewport.
2. **Fixed center pin:** the spot's `TypeBadge` (60px, with stem and ground shadow) absolutely centered; coordinates = map center (`map.getCenter()` on move). While the map is being dragged (`movestart`/`moveend` or drag events), apply the lift transform and enlarge the shadow, per the design.
3. **Chrome:** back button (cancel, returns without changes), title pill, hint chip (consider hiding it after first drag), bottom card with live coordinates formatted like the design (`56.1234° N · 10.2038° E`) — show GPS accuracy only if available from the caller, otherwise omit the "± m" hint — and the accent confirm CTA returning the center coordinates through the existing callback.
4. Pass the spot's type into the picker so the pin badge matches (extend props if the current contract lacks it; for new spots use the type selected in the add sheet, falling back to a generic badge).
5. Both themes + safe areas. Remove the old modal UI entirely.

# Mapbox Studio — custom light & dark styles for "Skovens Skatte"

Manual step for subtask 1.4. The app already reads `VITE_MAPBOX_STYLE_LIGHT` /
`VITE_MAPBOX_STYLE_DARK` from `.env` (falling back to stock `outdoors-v12` / `dark-v11`), so the
app works today — these two Studio styles replace the fallbacks with the design's earthy basemap.

## Color targets (extracted from the design file, `Skovens Skatte.dc.html` LIGHT/DARK constants)

| Design token | Role on the map | Light | Dark |
|---|---|---|---|
| `--map-bg` | Base background (bare ground, urban fill) | `#e6ddc6` | `#1c261b` |
| `--map-land` | Vegetation / landuse (forest, park, grass) | `#d3ddc0` | `#233020` |
| `--map-water` | Water (lakes, sea, rivers) | `#bcd0e0` | `#1f3540` |
| `--map-line` | Roads and contour lines | `#cabf9f` | `#31402c` |
| `--map-trail` | Trails / footpaths | `#c9a15f` | `#c9a24b` |

Supporting text/label tones (from the same palette, for the few labels we keep):

| Token | Light | Dark |
|---|---|---|
| `--ink` (primary label text) | `#23301f` | `#f0e9d6` |
| `--muted` (secondary label text) | `#8a8577` | `#9fae94` |
| Label halo (use `--bg`) | `#f4efe3` | `#1c261b` |

**Overall look:** calm and quiet. The design's map shows *only* land shapes, water, thin
contour/road lines, and one dashed trail — no POI icons, no bright colors, very few labels.

**Important functional requirement:** the calm look must NOT come at the cost of detail the app
depends on. Users relate their findings to where they stand in the forest, so **all small roads,
forest tracks, and walking trails must be visible** — including `track`, `service`, `path`,
`footway`, and `pedestrian` classes. Reduce *labels and POIs*, not the road/trail network itself.

## Steps (once per style — do Light first, repeat for Dark)

1. Go to [Mapbox Studio](https://studio.mapbox.com/) → **New style**. In the "Choose a starting
   point" dialog, **ignore the tiles in the top rows** (Standard, Faded, Monochrome, Light 2D,
   Dark 2D, Satellite) — those are built on the new *Mapbox Standard* architecture, which
   requires Mapbox GL JS v3, and the app runs GL JS v2.15 (`mapbox-gl` in `package.json`), so
   they would not render in the app. Instead, under **Other options**, open **Classic styles**
   (arrow button) and choose the **Outdoors** template. Outdoors already includes the full
   trail/track/path network, contour lines, and outdoor-tuned zoom ranges, which the app depends
   on; you "only" need to recolor and thin out labels.
2. Name it `skovens-skatte-light` (resp. `skovens-skatte-dark`).
3. In the style editor, use the style configuration's **Colors** tab first (set the base/greenspace
   colors near `--map-bg`/`--map-land`), then fine-tune per layer group in the **Layers** panel:

### Per-layer-group color mapping

Apply the **Light** column for the light style and the **Dark** column for the dark style:

| Studio layer group / layers | Property | Light | Dark |
|---|---|---|---|
| `land`, `background` | background-color / fill-color | `#e6ddc6` | `#1c261b` |
| `landuse`, `landcover`, `national-park` | fill-color, per class — see the class table below | — | — |
| `water`, `water-shadow`, `waterway` | fill-color / line-color | `#bcd0e0` | `#1f3540` |
| All `road-*` layers (motorway, primary, secondary, street, minor) | line-color | `#cabf9f` | `#31402c` |
| `road-minor` (contains forest tracks!) | **class-driven color**: Danish forest walking routes (e.g. "Gul Sti") are OSM `highway=track` → class `track` in this layer. Set Data field to `class`: `track` → trail gold (solid), fallback (`service` etc.) → road color. **Lower minzoom to ~11** so tracks show as soon as a forest fills the screen | track `#c9a15f`, rest `#cabf9f` | track `#c9a24b`, rest `#31402c` |
| `road-street-low` | line-color | `#cabf9f` | `#31402c` |
| ALL `road-*-case` + `road-path-bg`, `road-steps-bg` (casings/halo — wider lines drawn UNDER the road/path; do NOT color path-bg gold or the dash gaps fill in) | line-color, one flat color for all; match each casing's minzoom to its parent layer | `#e6ddc6` (or delete casings) | `#1c261b` (or delete casings) |
| `contour-line`, `contour-label` (included in the Outdoors template) | line-color, ~40% opacity | `#cabf9f` | `#31402c` |
| `road-path`, `road-steps`, `road-pedestrian` (footpaths) | line-color, dashed (`2, 8`), line-cap round — **keep ALL of these; lower minzoom to ~11–12** and bump line-width slightly (~+0.5) so trails stand out inside forests. Resulting walking language: solid gold = gravel track (road-minor `track`), dashed gold = footpath, parchment = ordinary road | `#c9a15f` | `#c9a24b` |
| `hillshade` (if present) | delete or set opacity ≤ 0.05 | — | — |
| `building`, `building-outline` | fill-color slightly darker than bg, e.g. `#ddd3ba` / opacity ~0.4 | see left | `#233020` / opacity ~0.4 |

### Landuse/landcover class colors (nuanced, foraging-aware)

The `landuse` and `landcover` fill colors are **class-driven expressions** (a swatch per landuse
class, possibly nested inside zoom stops). In the editor: open the layer → Color → if there are
two zoom stops with identical intent, delete one stop first (halves the work), then expand the
swatch strip and set each class condition. Paste hexes at full opacity — the originals are
semi-transparent `hsla()` tints, but the design is flat, so these ARE the final colors.

Instead of one flat green, use subtle tiers — they carry real foraging information (forest vs
scrub vs open grass, dunes vs fields) while staying calm. Differences are a few percent of
lightness; at zoom 11–13 the map must still read as one quiet landscape:

| Class | Light | Dark | Rationale |
|---|---|---|---|
| wood | `#c6d3af` | `#283626` | deepest green — forest is the primary terrain |
| scrub | `#cdd8b8` | `#253323` | between forest and open |
| park, grass, cemetery, pitch | `#d3ddc0` | `#233020` | base `--map-land` tone |
| agriculture, crop | `#dedec3` | `#222c1e` | green-cream hint — fields read as open land so forests pop (most of DK is farmland) |
| sand | `#e9ddb2` | `#2e2f20` | warm gold — dunes/beaches (sea buckthorn, rose hip terrain) |
| rock | `#ded8c8` | `#272b26` | desaturated gray |
| glacier | `#e2e6e2` | `#243030` | cool/pale (moot in DK) |
| airport, hospital, school, commercial_area, residential, facility/industrial | `#e6ddc6` | `#1c261b` | urban — same as base, keep quiet |
| Fallback value | `#e6ddc6` | `#1c261b` | base land |

Guardrails: use the SAME hex for a class wherever it appears (`landuse`, `landcover`,
`national-park` overlap — mismatches show as seams). All fills must stay lighter than the road
color (light theme) / darker than the trail gold (dark theme) so the line network keeps top
visual priority. Eyeball over the Thy coast (Østerild: dunes + plantation + fields in one view)
and nudge to taste.

### Labels & POIs (reduce density — the design is nearly label-free)

The editor's layer panel groups label layers at the top (they render topmost); the line/fill
layers to recolor are further down. Group names below match the Studio editor for classic
Outdoors:

| Editor group → layer | Action |
|---|---|
| Point of interest labels → `poi-label` | **Delete** |
| Transit, transit-labels → `airport-label`, `transit-label` | **Delete** |
| Transit, ferry-aerialway-labels → `ferry-aerialway-label` | **Delete** |
| Walking, cycling, etc. → `golf-hole-label` | **Delete** |
| Walking, cycling, etc. → `path-pedestrian-label` | **Keep** — draws trail names ("Gul Sti"), key for orientation. Text `#23301f` light / `#f0e9d6` dark (ink — muted gray is unreadable on the gold lines), halo `#f4efe3` / `#1c261b`, halo width ~1.5 |
| Road network, road-labels → `road-exit-shield`, `road-intersection` | **Delete** |
| Road network, road-labels → `road-label`, `road-number-shield` | Keep; road-label text `#23301f` light / `#f0e9d6` dark (labels render ON the line — ink + halo stays readable on gold/parchment), halo `#f4efe3` / `#1c261b`, halo width ~1.5 |
| Buildings, building-labels → `block-number-label`, `building-number-label`, `building-entrance` | **Delete** |
| Place labels → `continent-label`, `state-label` | **Delete** (irrelevant in Denmark) |
| Place labels → `settlement-*-label`, `country-label` | Keep; text `#23301f` light / `#f0e9d6` dark, halo as above. Optional: quieten `settlement-major/minor-label` by lowering each Size zoom stop 1–2 px (per layer — stops differ, so no bulk edit) |
| Natural features, natural-labels → all (`natural-*`, `water-*`, `waterway-label`) | Keep — forest & lake names matter for foraging; text `#8a8577` light / `#9fae94` dark |
| Terrain, terrain-labels → `contour-label` | Keep, muted (`#8a8577` / `#9fae94`) — elevation aids orientation |

Tip: Cmd-click (Ctrl-click) selects multiple layers so you can set a shared color property once —
useful for all `road-*` line layers, or a whole label group.

Tip: if a layer keeps its original tint after the configuration-level recolor, use *Select all
layers in group → color* and paste the hex directly; setting the style configuration's base
colors close to `--map-bg`/`--map-land` first saves per-layer edits.

4. Zoom around a real Danish forest (e.g. Silkeborg-skovene, Rold Skov, Gribskov) at zooms 6–17
   and sanity-check:
   - forests read as `--map-land` green, water is the muted blue, roads are thin parchment
     lines, and nothing bright (default Mapbox blues/oranges) remains;
   - **at zoom ~12–13 with a forest filling the screen, the forest tracks and walking trails are
     clearly visible** as gold dashed lines — this is the critical check, since users navigate to
     their finds by these small paths. If a track or path you know exists doesn't show, find its
     layer (click it in the map with the Outdoors base visible) and lower that layer's minzoom.
5. **Publish** the style (Publish button, make it public or keep private — the app's token can
   access your own private styles).
6. Copy the **style URL** (Share → Style URL, format `mapbox://styles/<username>/<style-id>`).

## Wire into the app

Add both URLs to `.env` (see `.env.example`):

```bash
VITE_MAPBOX_STYLE_LIGHT=mapbox://styles/<username>/<light-style-id>
VITE_MAPBOX_STYLE_DARK=mapbox://styles/<username>/<dark-style-id>
```

Restart `npm run dev`. The map (and the location picker) switch styles automatically with the
theme toggle / OS theme. Offline tile caching needs no changes — the Workbox rule caches all of
`api.mapbox.com`, which serves custom Studio styles the same way as stock ones.

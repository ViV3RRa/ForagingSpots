## Task

Systematically walk **every** screen and overlay in both themes (dev server + devtools mobile viewport; additionally iOS standalone if available): welcome, sign-in, permission screen, map (pins/clusters/locate/user dot/location chip), list, empty state, detail drawer (all gallery cases), photo lightbox, add/edit sheet (keyboard open too), location editor, filter sheet, delete confirm, offline banner, install sheet, update toast, boot splash.

For each, check and fix:

1. **Dark palette fidelity:** every surface/text/border uses tokens (hunt for hardcoded hex values with grep across `src/components/`), accent correctly resolves to gold in dark, `--pin-ring` visible against the dark map, scrims and shadows per `DARK`.
2. **Contrast:** body text and mono labels legible in both themes (spot-check WCAG AA for small text).
3. **Safe areas:** floating top bar, bottom toggle, FAB, sheets' bottom padding, lightbox close button — all respect notch/home-indicator insets.
4. **Theme switching live:** toggle while each overlay is open — no stale colors (e.g. map style swap mid-sheet), no flash of wrong theme on reload.
5. Keyboard-over-sheet behavior in add/edit and sign-in.

Fix everything you find directly; keep a short list of anything intentionally deferred at the bottom of this file's PR/commit message.

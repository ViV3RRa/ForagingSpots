## Task

1. **App icons:** generate new icons from the design concept — rounded-square, brand-green or cream background with the three-badge cluster. Build an SVG master (recreate with the existing PNGs positioned per the design, or vector approximations), then export/rasterize 192×192, 512×512, and a maskable 512 (safe zone!) into `public/`. Add an `apple-touch-icon` (180×180). Use a script or manual canvas/sharp step — check what tooling is available; committing generated PNGs is fine.
2. **Manifest (`vite.config.ts`):** `theme_color: '#2f4a32'`, `background_color: '#f4efe3'`, verify `name`/`short_name` ("Skovens Skatte"), icon entries point at the new files, `display: 'standalone'` retained.
3. **`index.html`:** apple-touch-icon link; `theme-color` metas with `media="(prefers-color-scheme: …)"` variants (light `#f4efe3`, dark `#1c261b`) as the static fallback — the ThemeContext from 1.1 updates it dynamically; make sure the two don't fight (dynamic wins after hydration).
4. **In-app boot splash (optional, keep trivial):** native splash comes from manifest colors + icon. If AuthContext restore causes a visible blank flash at launch, add a minimal brand-colored boot div (wordmark + dots per the design) shown until auth state resolves. If there's no flash, skip and note it.
5. Rebuild and verify precache picks up new icons.

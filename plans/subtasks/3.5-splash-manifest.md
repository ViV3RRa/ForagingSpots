## Task

> **Scope change (2026-07-09):** app icons are no longer generated here. The design's final
> monogram icon set (Spectral "S" on terracotta, `App Icon - Skovens Skatte.dc.html`) is already
> committed in `public/app-icon/`, and pointing favicon/apple-touch/manifest icon entries at it
> is subtask **3.7** (`3.7-app-icon-replacement.md`). This subtask owns manifest colors/name,
> theme-color metas, and the optional boot splash. If 3.7 has not landed yet, leave the icon
> `src` entries as you find them.

1. **Manifest (`vite.config.ts`):** `theme_color: '#2f4a32'` (or `#f4efe3` — match chrome), `background_color: '#f4efe3'`, verify `name`/`short_name` ("Skovens Skatte"), `display: 'standalone'` retained.
2. **`index.html`:** `theme-color` metas with `media="(prefers-color-scheme: …)"` variants (light `#f4efe3`, dark `#1c261b`) as the static fallback — the ThemeContext from 1.1 updates it dynamically; make sure the two don't fight (dynamic wins after hydration).
3. **In-app boot splash (optional, keep trivial):** native splash comes from manifest colors + icon. If AuthContext restore causes a visible blank flash at launch, add a minimal brand-colored boot div (wordmark + dots per the design) shown until auth state resolves. If there's no flash, skip and note it.
4. Rebuild and verify the manifest output.

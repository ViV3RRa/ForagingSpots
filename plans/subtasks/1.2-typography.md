# Subtask 1.2 — Typography (self-hosted fonts + type scale)

## Context

You are implementing one subtask of the "Skovens Skatte" UI redesign migration. Before starting:

1. Read `plans/new-design-migration.md` — the master plan (design language, token tables, scope, screen-to-component map). This file is **Phase 1, subtask 1.2**.
2. Read `CLAUDE.md` for project conventions and commands.

The app is a Danish foraging-spots PWA (React 19 + TypeScript + Vite, Tailwind CSS, shadcn/ui). It must work offline, so fonts are self-hosted — no runtime Google Fonts CDN.

**Depends on:** subtask 1.1 (tokens/theme) being merged.

- **For this subtask, study:** the Google Fonts `<link>` in the `<helmet>` block (exact families, weights, italics) and how the three families are used across the screen markup — heading sizes/weights, body sizes, and the recurring Space Mono micro-label pattern (font-size ~11px, `letter-spacing:.1–.16em`, `text-transform:uppercase`).

Font roles in the design:
- **Spectral** (serif): headings, buttons, editorial/body accents, italic species names and hints.
- **Work Sans** (sans): default body/UI text.
- **Space Mono**: coordinates, timestamps, uppercase section labels.

## Task

1. Install `@fontsource/spectral`, `@fontsource/work-sans`, `@fontsource/space-mono`. Import exactly the weights/styles the design's font link requests (check it — Spectral needs italics too) in `src/main.tsx`.
2. `tailwind.config.js`: set `fontFamily` so `font-serif` → Spectral, `font-sans` → Work Sans (make this the default body family), `font-mono` → Space Mono, each with appropriate fallback stacks.
3. Update the base typography layer in `src/styles/tokens.css`: headings use Spectral 600 with the design's sizes (inspect h1/h2/h3 usage in the mock: ~44px welcome title, 34px screen titles, 23–26px sheet titles, 19px list-row titles), body in Work Sans, and remove the old Inter-based rules.
4. Add a reusable way to apply the mono micro-label style (used by nearly every later subtask) — either a Tailwind component class (e.g. `.label-mono` in `@layer components`) or leave a note that subtask 1.3 creates a `MonoLabel` component (coordinate: check if `src/components/ui/MonoLabel` already exists).
5. Verify offline: run `npm run build` and confirm the woff2 files are emitted as hashed assets and matched by the Workbox `globPatterns` in `vite.config.ts` (extend patterns if woff2 is not covered).

- `npm run build` + `npm run lint` pass.
- Headings render in Spectral, body in Work Sans; no requests to fonts.googleapis.com anywhere.
- Font files are precached by the service worker (verify in the generated `sw.js`/precache manifest or via `npm run preview` offline).
- The mono-label style is available for later subtasks.

# Subtask 3.9 — Satellite map mode

## Context

You are implementing one subtask of the "Skovens Skatte" UI redesign migration. Before starting:

1. Read `plans/new-design-migration.md` — the master plan (design language, token tables, scope, screen-to-component map). This file is **Phase 3, subtask 3.9** (added 2026-07-09).
2. Read `CLAUDE.md` for project conventions and commands.

The design gained a **satellite map mode**: a circular "layers" toggle button on the map view
that swaps the basemap between the earthy base style and satellite imagery. Today the app has no
satellite anything — the map style is purely theme-keyed: `getMapStyle(theme)` in
`src/utils/mapbox.ts`, consumed by `MapView.tsx` (`mapStyle={getMapStyle(theme)}`) and
`LocationEditorScreen.tsx`.

**Decisions (agreed 2026-07-09):**

- The real basemap is a **custom Mapbox Studio satellite style** (single style, theme-independent
  — the design's satellite view is identical in light and dark), analogous to the 1.4 styles.
  The design mock's satellite layer is a hand-drawn SVG; treat it as art direction, not as
  something to replicate literally.
- Mode is **session-only**: it persists across Kort/Liste switches and sheet open/close within a
  session, but the app always boots in base mode. **No localStorage.**

**Depends on:** 1.1–1.4 (tokens, fonts, primitives, base map styles), 2.3/2.4 (app chrome, map
view). **Interacts with:** 2.12 (toggle hidden on the map-error screen), 2.14 (compass shares the
right edge), 3.8 (toggle stays visible in the no-location state — unlike the locate button).

## Mapbox Studio — satellite style (manual step)

Analogous to `plans/subtasks/1.4-mapbox-studio-instructions.md` (read it — its constraints and
label-thinning tables apply here too):

1. **New style → Classic styles → Satellite Streets** template. The Standard-architecture tiles
   in the top rows require GL JS v3; the app runs `mapbox-gl` v2.15, so Classic only. Satellite
   Streets (not plain Satellite) is the starting point because it layers the full vector
   road/trail network over the imagery — which the design's satellite trail shows and foragers
   depend on.
2. Name it `skovens-skatte-satellite`. One style serves both themes.
3. Thin it to the design's calm look using 1.4's label tables: delete POI, transit, shield,
   golf, and building labels; keep `path-pedestrian-label`, natural/water labels, and settlement
   labels — imagery is dark, so use light ink `#f0e9d6` text with a dark halo (`#1c261b`,
   width ~1.5) everywhere.
4. Roads: thin, desaturated light lines (they must not shout over the imagery). Tracks and
   footpaths (`road-minor` class `track`, `road-path`, `road-steps`, `road-pedestrian`): the
   design's satellite trail treatment — light gold `#d8c793`, dashed, round caps — with the same
   minzoom lowering to ~11–12 as 1.4 so forest tracks appear as soon as a forest fills the
   screen. Same functional requirement as 1.4: **all small roads, forest tracks, and walking
   trails must be visible**; delete casings or match them to the imagery (no light halos).
5. Sanity-check over a Danish forest (Silkeborg, Rold, Gribskov) at zooms 6–17: trails legible
   over the canopy, no bright default colors, labels sparse.
6. Publish, copy the style URL, and set it in `.env`:

   ```bash
   VITE_MAPBOX_STYLE_SATELLITE=mapbox://styles/<username>/<satellite-style-id>
   ```

The app must work before this style exists — fall back to stock
`mapbox://styles/mapbox/satellite-streets-v12` (Classic, GL JS v2-compatible), matching the
existing fallback pattern in `src/utils/mapbox.ts`.

## Task

1. **Style plumbing.** In `src/utils/mapbox.ts`, add the satellite style:
   `VITE_MAPBOX_STYLE_SATELLITE` with the stock fallback above, and extend the style lookup so
   callers can resolve by mode — e.g. `getMapStyle(theme, mode: 'base' | 'satellite' = 'base')`
   where satellite ignores the theme. Document the new env var in `.env.example`.
2. **Mode state.** Add `mapMode: 'base' | 'satellite'` state in `MainMapScreen.tsx` (plain
   `useState` — session-only by construction; it survives Kort/Liste switches and sheets because
   `MainMapScreen` stays mounted). Pass it to `MapView`, which resolves
   `mapStyle={getMapStyle(theme, mapMode)}`.
3. **Toggle button.** Render the design's 52px circular layers button on the map view, top-right
   under the floating top bar, respecting safe-area insets like the rest of the chrome. Place it
   wherever fits the existing chrome split best (the locate button lives in `MapView.tsx`; the
   chip in `MainMapScreen.tsx`) — visibility gates: map view only, hidden while any sheet is
   open and while the map-error card shows, **still rendered in the no-location state**.
   Inactive/active colors per the table; `aria-label` + `title` = `"Satellit"` (base) /
   `"Kort"` (satellite). No entrance/switch animation.
4. **Style-switch integrity.** Verify the react-map-gl style swap doesn't break overlays:
   markers/clusters are DOM-based and survive, but confirm viewport, user dot, and the 2.14
   compass behave across a base ⇄ satellite ⇄ theme-toggle sequence. If the existing
   `onError` handling (2.12) fires on a failed satellite style load, the map-error card must
   behave exactly as it does for a failed base style.
5. **Scope guard.** `LocationEditorScreen` and any other map surface keep the base style — only
   the main map toggles.

## Acceptance criteria

- `npm run build` + `npm run lint` pass.
- Map view shows the layers toggle top-right below the floating bar: 52px circle, surface bg +
  brand icon (base mode), correct in both themes.
- Tapping it swaps the basemap to satellite imagery (stock fallback until the Studio style is
  published and set in `.env`) and inverts the button (brand bg + brand-ink icon; gold-on-ink
  inversion in dark theme). Tapping again returns to the theme-correct base style.
- `aria-label` reads "Satellit" in base mode, "Kort" in satellite mode.
- Mode survives switching Kort → Liste → Kort and opening/closing the detail drawer, add sheet,
  and filter sheet; a full reload boots back in base mode (no persistence).
- Toggle is hidden while any sheet is open and on the map-error screen; it **is** visible in the
  no-location state (DevTools → Sensors → "Location unavailable") alongside the amber badge,
  with the locate button absent per 3.8.
- Toggling the theme while in satellite mode leaves the imagery unchanged; switching back to
  base lands on the correct light/dark base style.
- Pins, clusters, user-location dot, location chip, FAB, locate, and compass render unchanged
  over satellite imagery (`--pin-ring` stays theme-keyed).
- The location editor still opens with the base style regardless of the main map's mode.
- Offline: satellite tiles viewed online render again offline (the existing Workbox
  `api.mapbox.com` CacheFirst rule covers them; verify, no config change expected).

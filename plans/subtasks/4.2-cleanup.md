# Subtask 4.2 — Dead-code and style cleanup

## Context

You are implementing one subtask of the "Skovens Skatte" UI redesign migration. Before starting:

1. Read `plans/new-design-migration.md` — the master plan. This file is **Phase 4, subtask 4.2** — run after all Phase 2/3 subtasks (and ideally 4.1).

The migration replaced the old green/material design; this subtask removes what it left behind. **Design lookup is not needed for this subtask** (the design link is in the master plan if reference is required).

## Task

1. **Old tokens/styles:** remove any remaining old-palette CSS variables, oklch leftovers, Inter font references, old keyframes, and unused Tailwind config entries. Grep for the old colors (`#4CAF50`, `#9CCC65`, `#8D6E63`, `#2E7D32`, `green-50`, `green-100`, etc.).
2. **Unused components:** verify and remove superseded components — candidates: the per-type SVG icon components in `src/components/icons/*.tsx` (superseded by PNG-based `getForagingSpotConfig`; **verify zero imports first**), `src/components/FilterButton.tsx` if absorbed into the chrome (2.3), `src/components/figma/ImageWithFallback.tsx` if unused, `IconShowcase.tsx` (keep only if the team wants it as a living style reference — check with git history/master plan 1.3 which repurposed it; default: keep, but exclude from production paths).
3. **Unused shadcn/ui components:** `src/components/ui/` has 40+ components; remove ones with zero imports (verify with grep/ts-prune-style checking; be conservative — only delete clearly unimported files).
4. **Dependencies:** remove npm packages that are no longer imported anywhere (verify carefully).
5. Run `npm run lint` and `npm run build`; fix all warnings/errors introduced by removals. Do a final grep for `TODO`/`FIXME` added during the migration and resolve or ticket them.

## Files

- Deletions across `src/components/`, `src/styles/`, `tailwind.config.js`, `package.json`.

You are implementing one subtask of the "Skovens Skatte" UI redesign migration. Before starting:

## Task

1. **Rewrite `src/styles/tokens.css`:**
   - Replace the current green/oklch palette. Define the design's LIGHT palette under `:root` and DARK under `.dark`.
   - Keep the CSS-variable names consumed by Tailwind/shadcn (`--background`, `--foreground`, `--primary`, `--accent`, `--border`, `--card`, etc.) but map them onto the new values (e.g. `--background` → design `--bg`, `--primary` → design `--accent`, border → design `--line`).
   - Additionally expose the design-specific tokens as first-class variables with their design names: `--brand`, `--brand-ink`, `--accent-ink`, `--ink`, `--ink2`, `--muted`, `--faint`, `--line`, `--line2`, `--mono`, `--pin-ring`, `--map-bg`, `--map-land`, `--map-water`, `--map-line`, `--map-trail`, `--shadow`, `--pulse` — both themes.
   - Preserve the existing iOS safe-area utility classes (`.safe-area-top` etc.).
2. **Animations:** add the design's `ss-pulse`, `ss-rise`, `ss-fade` keyframes as reusable Tailwind animations (`tailwind.config.js` keyframes/animation entries) and/or in `tokens.css`.
3. **Theme infrastructure:** create a `ThemeContext` (new file, e.g. `src/contexts/ThemeContext.tsx`) providing `{ theme: 'light'|'dark', preference: 'light'|'dark'|'system', setPreference(...) }`:
   - Resolution: manual override from localStorage (key e.g. `ss-theme`) if set, otherwise `prefers-color-scheme`.
   - Applies/removes the `.dark` class on `<html>`; subscribes to system-theme changes while preference is `system`.
   - Dynamically updates `<meta name="theme-color">` (light `#f4efe3`, dark `#1c261b`).
   - Wrap the app with the provider in `src/App.tsx` (or `main.tsx`). The UI toggle itself is added later in subtask 2.3 — only the context is needed now.

## Files

- `tailwind.config.js` (colors reference the variables already; add keyframes/animations)

- `index.html` (theme-color meta if adjustments needed)

## Constraints

- Do not restyle individual components in this subtask — the app should render with recolored-but-unchanged layout.
- Do not break existing Tailwind utility usage (`bg-background`, `text-foreground`, `border-border`, …).

## Acceptance criteria

- `npm run build` and `npm run lint` pass; app runs with `npm run dev`.
- All screens render with the new palette (layout unchanged).
- Adding `.dark` to `<html>` in devtools shows the complete dark palette.
- With no stored preference, theme follows the OS setting live; a stored override wins and survives reload.
- `ss-pulse`, `ss-rise`, `ss-fade` animations are available as utilities/classes.
